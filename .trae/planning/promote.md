## MVP风险应对预案索引

### 常见风险快速响应指南

#### 场景1: 数据库连接失败

**症状：** 应用启动报错 `Cannot create PoolableConnectionFactory`

**可能原因：**
1. MySQL服务未启动
2. 连接配置错误（URL、用户名、密码）
3. 数据库不存在
4. 端口冲突

**排查步骤：**
1. 检查MySQL服务状态: `net start mysql` 或 `systemctl status mysql`
2. 验证连接配置: `application.yml` 中的 `spring.datasource.*`
3. 测试连接: 使用Navicat/DBeaver等工具连接
4. 检查schema.sql是否已执行

**解决方案：**
- 启动MySQL服务
- 修正配置参数
- 执行初始化脚本

**预防措施：**
- 编写启动检查脚本
- 在README中明确环境要求

---

#### 场景2: Redis连接超时

**症状：** 应用启动慢或报错 `Unable to connect to Redis`

**可能原因：**
1. Redis服务未启动
2. 配置错误（host/port/password）
3. 防火墙阻止
4. Redis内存不足

**排查步骤：**
1. 检查Redis服务: `redis-cli ping` 应返回 `PONG`
2. 验证配置: `spring.data.redis.*`
3. 检查防火墙: 确保6379端口开放
4. 查看Redis日志

**解决方案：**
- 启动Redis: `redis-server`
- 修正配置
- 调整防火墙规则

**MVP降级方案：**
如Redis暂时不可用，可临时禁用缓存相关功能（Token黑名单等），使用内存Map替代

---

#### 场景3: JWT Token验证失败

**症状：** 登录成功但访问接口返回401

**可能原因：**
1. Token过期（默认2小时）
2. Secret Key不匹配（前后端配置不一致）
3. Token格式错误（Bearer前缀缺失）
4. 用户被禁用或锁定

**排查步骤：**
1. 检查Token是否过期: 解码JWT查看exp字段
2. 对比Secret: 确保前后端jwt.secret一致
3. 检查请求头: `Authorization: Bearer <token>`
4. 查询用户状态: status=1, is_locked=false

**解决方案：**
- 刷新Token或重新登录
- 统一配置文件
- 修正前端请求拦截器

#### 场景4: AES加密解密异常

**症状：** 存储Key时或读取时报错 `AES加密/解密失败`

**可能原因：**
1. 密钥长度不足（需32字符/256位）
2. 加密算法不匹配（GCM vs ECB）
3. IV向量丢失或损坏
4. 数据库字段类型不匹配（TEXT vs VARCHAR）

**排查步骤：**
1. 检查密钥长度: `aes.secret-key` 必须32字符
2. 确认算法: 加密和解密使用相同算法和模式
3. 验证IV存储: GCM模式需要存储IV（通常拼接在密文前）
4. 检查字段类型: schema.sql中使用TEXT

**解决方案：**
- 修正密钥长度
- 统一加密实现
- 检查数据完整性

**数据恢复：**
如果密钥更换导致旧数据无法解密：
1. 备份旧数据
2. 使用旧密钥批量解密
3. 使用新密钥重新加密
4. 更新数据库记录

#### 场景5: 前后端联调CORS错误

**症状：** 浏览器控制台报跨域错误 `Access-Control-Allow-Origin`

**可能原因：**
1. 后端未配置CORS
2. Vite代理配置错误
3. 前端请求地址错误

**排查步骤：**
1. 检查浏览器Network面板: 查看实际请求地址
2. 检查Vite配置: `vite.config.js` 的proxy设置
3. 检查后端CORS配置: 是否添加了CorsFilter或@CrossOrigin

**解决方案：**

**前端方案（推荐）- Vite代理：**
```javascript
// vite.config.js
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }
}
```

**后端方案 - 全局CORS：**
```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
                .allowedOrigins("http://localhost:3000")
                .allowedMethods("GET", "POST", "PUT", "DELETE")
                .allowedHeaders("*")
                .allowCredentials(true);
    }
}
```
