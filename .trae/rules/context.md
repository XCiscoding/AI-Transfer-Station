# 上下文治理与问题单规则

## 文档类型
Project Rule

## 目标
控制上下文污染，确保每轮执行只基于当前合法任务和最小必要事实。

## 上下文优先级
本系统事实来源优先级如下：
1. 项目总经理最近一次合法调度
2. 当前任务单 / 问题单
3. summary.md
4. task.md
5. spec.md
6. progress/ 与相关日志
7. 其他历史对话

历史对话优先级最低，不得覆盖显式文档事实。

## 上下文使用规则
1. 每轮执行只读取完成本轮任务所需的最小上下文。
2. 状态切换、角色切换、测试失败、Debug 回退后，必须重新整理上下文，不得沿用整段旧对话直接续跑。
3. 若共享文档、任务单、当前请求互相冲突，必须停止并返回项目总经理裁定。
4. 不得用主观猜测补全文档缺失事实。

## 问题单规则
测试失败或发现缺陷时，必须创建或更新问题单，至少包含：
- 问题标题
- 所属模块
- 复现步骤
- 预期结果
- 实际结果
- 证据
- 当前状态
- 已尝试次数

## 熔断规则
1. 同一问题连续两次失败，必须重新定位。
2. 连续三次失败，必须升级给项目总经理重点审查。
3. 连续四次失败，进入 BLOCKED，等待更强证据或人工干预。
