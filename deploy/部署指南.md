```
│  │       │              │          │    │
```

## 二、购买云服务器

### 2.1 推荐配置

| 配置项 | 推荐规格 | 说明 |
|--------|----------|------|
| CPU | 2核 | 足够运行所有服务 |
| 内存 | 4GB | 最低要求，建议4G以上 |
| 带宽 | 3-5M | 访问速度有保障 |
| 系统盘 | 40GB+ | 存放代码和数据库 |
| 操作系统 | CentOS 7/8 或 Ubuntu 20.04+ | 推荐 CentOS 7.9 |

### 2.2 购买步骤（以阿里云为例）

1. 访问 [阿里云官网](https://www.aliyun.com/)
2. 搜索"云服务器ECS" → 点击"立即购买"
3. 选择配置：
   - 计费方式：包年包月（长期使用）或按量付费（测试）
   - 地域：选择离你最近的
   - 实例规格：2核4G（共享标准型s6 或 突发性能实例t6）
   - 镜像：CentOS 7.9 64位
   - 带宽：3Mbps
   - 安全组：先不设置，后面单独配置
4. 设置登录密码（记住这个密码！）
5. 确认订单并支付

💡 **省钱提示**：新用户首年通常有优惠，约 200-400元/年

## 三、配置安全组

### 3.1 什么是安全组？

安全组 = 防火墙规则，决定哪些端口可以被外部访问

### 3.2 需要开放的端口

| 端口 | 用途 | 必须开放？ |
|------|------|-----------|
| 22 | SSH远程连接 | ✅ 必须 |
| 80 | HTTP网页访问 | ✅ 必须 |
| 443 | HTTPS（暂不需要） | ❌ 可选 |

### 3.3 配置步骤

**阿里云为例**：

1. 进入 [ECS控制台](https://ecs.console.aliyun.com/)
2. 找到你的服务器 → 点击"安全组"
3. 点击"配置规则"
4. 点击"入方向" → "手动添加"
5. 添加以下规则：

```
规则1：
- 授权策略：允许
- 优先级：1
- 协议类型：自定义TCP
- 端口范围：22/22
- 授权对象：0.0.0.0/0
- 描述：SSH远程连接

规则2：
- 授权策略：允许
- 优先级：1
- 协议类型：自定义TCP
- 端口范围：80/80
- 授权对象：0.0.0.0/0
- 描述：HTTP网页访问
```

⚠️ **注意**：`0.0.0.0/0` 表示允许所有IP访问，生产环境建议限制为你的IP

## 四、连接服务器

### 4.1 安装 FinalShell

1. 访问 [FinalShell官网](http://www.hostbuf.com/)
2. 下载对应系统的版本（Windows/Mac/Linux）
3. 安装并打开

### 4.2 创建连接

1. 点击左上角 "文件夹+" 图标 → "SSH连接"
2. 填写连接信息：
   ```
   名称：AI调度中心服务器
   主机：你的服务器公网IP（如 123.45.67.89）
   端口：22
   用户名：root
   密码：你购买时设置的密码
   ```
3. 点击"确定"
4. 双击连接名称，首次连接选择"接受并保存"

### 4.3 连接成功标志

看到类似下面的提示表示成功：
```
[root@iZbp123456789 ~]# 
```

## 五、上传项目文件

### 5.1 需要上传哪些文件？

从本地项目文件夹中，上传以下内容到服务器：

```
必须上传的文件：
├── backend/              # 后端代码文件夹
│   ├── src/
│   ├── pom.xml
│   └── ...
├── frontend/             # 前端代码文件夹
│   ├── src/
│   ├── package.json
│   └── ...
└── deploy/               # 部署配置文件夹
    ├── docker-compose.all-in-one.yml
    ├── Dockerfile.backend
    ├── Dockerfile.frontend
    ├── nginx.conf
    ├── finalshell-deploy.sh
    └── .env.example
```

### 5.2 上传步骤

**方法：使用 FinalShell 文件管理器**

1. 在 FinalShell 左侧找到"文件管理器"
2. 本地：找到你的项目文件夹
3. 远程：进入 `/opt` 目录（命令：`cd /opt`）
4. 创建项目目录：`mkdir aikey`
5. 进入目录：`cd aikey`
6. 拖拽上传：
   - 将 `backend` 文件夹拖到远程 `/opt/aikey/`
   - 将 `frontend` 文件夹拖到远程 `/opt/aikey/`
   - 将 `deploy` 文件夹拖到远程 `/opt/aikey/`

### 5.3 验证上传

在 FinalShell 终端执行：
```bash
cd /opt/aikey
ls -la
```

应该看到：
```
drwxr-xr-x 5 root root 4096 Jan 15 10:00 .
drwxr-xr-x 3 root root 4096 Jan 15 09:59 ..
drwxr-xr-x 6 root root 4096 Jan 15 10:00 backend
drwxr-xr-x 6 root root 4096 Jan 15 10:00 deploy
drwxr-xr-x 6 root root 4096 Jan 15 10:00 frontend
```

## 六、执行部署

### 6.1 进入部署目录

```bash
cd /opt/aikey/deploy
```

### 6.2 运行一键部署脚本

```bash
bash finalshell-deploy.sh
```

### 6.3 部署过程说明

脚本会自动执行以下步骤：

| 步骤 | 耗时 | 说明 |
|------|------|------|
| 1. 检查环境 | 5秒 | 检查系统版本和权限 |
| 2. 安装Docker | 2-5分钟 | 自动安装Docker和Docker Compose |
| 3. 准备目录 | 5秒 | 创建必要的文件夹 |
| 4. 配置环境变量 | 5秒 | 自动生成数据库密码 |
| 5. 构建镜像 | 10-20分钟 | 编译前端和后端代码 |
| 6. 启动服务 | 1-2分钟 | 启动所有容器 |
| 7. 健康检查 | 30秒 | 验证服务是否正常 |

**总耗时：约15-30分钟**

看到以下输出表示成功：
```

📱 访问地址：

   地址：localhost:3306
```

## 八、常见问题

### Q1: 部署脚本运行失败

**现象**：脚本报错退出

**解决**：
```bash
# 1. 检查是否在 deploy 目录
cd /opt/aikey/deploy
pwd  # 应该显示 /opt/aikey/deploy

# 2. 检查文件是否存在
ls -la finalshell-deploy.sh

# 3. 给脚本执行权限
chmod +x finalshell-deploy.sh

# 4. 重新运行
bash finalshell-deploy.sh
```

### Q2: Docker安装失败

**现象**：提示 "Failed to install Docker"

**解决**：
```bash
# 手动安装Docker
curl -fsSL https://get.docker.com | sh
systemctl start docker
systemctl enable docker
docker --version  # 验证安装

# 然后重新运行部署脚本
bash finalshell-deploy.sh
```

### Q3: 端口被占用

**现象**：提示 "port 80 is already allocated"

**解决**：
```bash
# 查看占用80端口的进程
netstat -tlnp | grep 80

# 停止占用端口的进程（如nginx、apache）
systemctl stop nginx
systemctl stop httpd

# 或者修改docker-compose使用其他端口
# 编辑 docker-compose.all-in-one.yml
# 将 "80:80" 改为 "8080:80"
```

### Q4: 数据库连接失败

**现象**：后端日志显示数据库连接错误

**解决**：
```bash
# 查看后端日志
docker compose -f docker-compose.all-in-one.yml logs backend

# 等待数据库完全启动（首次启动需要30-60秒）
sleep 60

# 重启后端服务
docker compose -f docker-compose.all-in-one.yml restart backend
```

### Q5: 页面显示404

**现象**：能访问但显示404错误

**解决**：
```bash
# 检查前端容器状态
docker compose -f docker-compose.all-in-one.yml ps frontend

# 查看前端日志
docker compose -f docker-compose.all-in-one.yml logs frontend

# 重启前端
docker compose -f docker-compose.all-in-one.yml restart frontend
```

### Q6: 无法访问网站

**现象**：浏览器打不开页面

**排查步骤**：
1. 检查安全组是否开放80端口
2. 检查服务器防火墙：
   ```bash
   systemctl status firewalld
   # 如果开启，添加规则：
   firewall-cmd --permanent --add-port=80/tcp
   firewall-cmd --reload
   ```
3. 检查服务状态：
   ```bash
   docker compose -f docker-compose.all-in-one.yml ps
   ```

## 九、日常维护

### 9.1 常用命令

```bash
# 进入项目目录
cd /opt/aikey/deploy

# 查看所有服务状态
docker compose -f docker-compose.all-in-one.yml ps

# 查看日志（实时）
docker compose -f docker-compose.all-in-one.yml logs -f

# 查看特定服务日志
docker compose -f docker-compose.all-in-one.yml logs -f backend
docker compose -f docker-compose.all-in-one.yml logs -f frontend

# 重启所有服务
docker compose -f docker-compose.all-in-one.yml restart

# 重启特定服务
docker compose -f docker-compose.all-in-one.yml restart backend

# 停止所有服务
docker compose -f docker-compose.all-in-one.yml down

# 停止并删除数据（谨慎！）
docker compose -f docker-compose.all-in-one.yml down -v
```

### 9.2 备份数据

```bash
# 备份数据库
docker exec aikey-mysql mysqldump -u root -p ai_key_management > backup_$(date +%Y%m%d).sql

# 输入密码后，备份文件会在当前目录生成
```

### 9.3 更新代码后重新部署

```bash
cd /opt/aikey/deploy

# 1. 停止服务
docker compose -f docker-compose.all-in-one.yml down

# 2. 重新构建（代码有更新时）
docker compose -f docker-compose.all-in-one.yml up -d --build

# 3. 查看状态
docker compose -f docker-compose.all-in-one.yml ps
```

### 9.4 查看资源使用

```bash
# 查看容器资源使用
docker stats

# 查看服务器整体资源
df -h  # 磁盘
free -h  # 内存
top  # CPU和进程
```
